# Skill Switch 交互重设计

## 核心理念

每个目标目录（对应不同开发工具）独立绑定一个主题。

**使用场景**：
- `~/.claude/skills` → "全量"主题（Claude Code 用全部技能）
- `~/.opencode/skills` → "开发"主题（OpenCode 只用开发相关技能）
- `~/.codex/skills` → "演示"主题（Codex 只用演示技能）

## 当前问题

### 1. 概念层级错乱

用户需要理解 4 个概念才能使用：
- **仓库目录**：技能文件存放在哪
- **目标目录**：符号链接创建到哪
- **主题**：一组技能的集合
- **符号链接**：底层实现细节

这些概念之间的关系不直观。用户只想"切换技能组合"，却被迫理解底层实现。

### 2. 核心操作被隐藏

切换主题是核心功能，但它的位置是：
```
Dashboard → 目录状态区域 → 某个目录行 → 小按钮"切换" → 下拉选择主题
```

4 次点击才能完成一次切换。

### 3. 页面职责混乱

**Dashboard（主题页）** 同时承担两个职责：
- 显示目录状态（实现细节）
- 管理主题列表（用户功能）

**技能页** 只是一个只读列表，没有实际作用。

### 4. 初始化过于复杂

3 步向导：仓库目录 → 预览技能 → 目标目录

用户第一次使用就被要求配置两个目录，不理解为什么要配两个。

---

## 设计目标

1. **概念简化**：用户只需要理解"主题"，不需要知道目录、符号链接
2. **操作直达**：每个目录独立切换主题，≤ 2 次点击
3. **页面职责单一**：所有设置都在主页进行

---

## 新设计方案

### 页面结构

```
┌─────────────────────────────────────────────────────────────┐
│  Skill Switch                                               │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  技能仓库                                                    │

│  │  skill-a  skill-b  skill-c  ...                         │ │

│                                                              │

│  │  [编辑]   │  │  [编辑]   │  │  [编辑]   │                   │
│  └──────────┘  └──────────┘  └──────────┘                   │
└─────────────────────────────────────────────────────────────┘
```

### 核心交互

**主页（唯一页面）**：
- 顶部：技能仓库地址（可编辑）
- 中部：每个目标目录独立显示，每个目录有自己的主题下拉选择
- 底部：主题管理卡片网格，可新建、编辑、删除主题

**操作流程**：
1. 打开页面，看到每个目录及其当前主题
2. 点击目标目录的下拉框，选择主题
3. 完成

**2 次点击完成切换。**

### 技能仓库

显示当前技能仓库地址，点击"编辑"可修改。

### 目录卡片

每个目标目录显示：
- 目录路径
- 当前主题名称 + 技能数量
- 技能列表（标签形式）
- 主题切换下拉框
- 编辑/删除按钮

### 主题卡片

```
┌────────────────────────┐
│  开发                   │
│  3 个技能               │
│                        │
│  [编辑]  [删除]         │
└────────────────────────┘
```

- **编辑**：展开内联编辑（重命名、增删技能）
- **删除**：确认后删除

### 设置页（隐藏高级功能）

点击右上角"设置"进入：

```

│  ┌─────────────────────────────────────────────┐ │
│  │  ~/Documents/github-skills         [浏览]   │ │
│  └─────────────────────────────────────────────┘ │
│  包含所有技能文件夹的目录                          │
│                                                  │
│  目标目录                                        │
│  ┌─────────────────────────────────────────────┐ │
│  │  ~/.claude/skills                  [浏览]   │ │
│  └─────────────────────────────────────────────┘ │
│  主题切换时会在此目录创建符号链接                   │

│                                                  │
└─────────────────────────────────────────────────┘
```

```
┌─────────────────────────────────────────────────┐

│  请选择技能仓库目录：                              │
```

**关键改动**：
- 合并为 1 页，不需要"下一步"
- 目标目录有默认值，用户可以不填
- 扫描是实时的，输入路径后自动扫描

## 数据模型变更

不需要改数据模型，只改 UI 交互。

现有的 `Config` 结构完全够用：
```typescript
interface Config {
  store: string;
  targets: TargetConfig[];
  themes: Record<string, string[]>;
}

interface TargetConfig {
  path: string;
  theme: string;
}
```

## API 变更

不需要改 API。现有接口完全够用。

## 页面重组

### 删除

- `web/src/pages/Skills.tsx` — 技能列表页面，无实际作用
- `web/src/pages/Settings.tsx` — 设置页面，所有设置移至主页

### 重写

- `web/src/pages/Dashboard.tsx` — 新主页（包含所有设置）
- `web/src/pages/Setup.tsx` — 简化初始化向导

### 新增

- 无

## 交互细节

### 目录主题切换

每个目录独立切换主题，使用下拉框：
- 选择即生效，不需要确认
- 切换后自动刷新状态

### 当前主题高亮

目录卡片显示当前绑定的主题：
- 主题名称加粗
- 显示技能数量
- 技能列表以标签形式展示

### 主题编辑

点击"编辑"按钮，卡片展开内联编辑：
- 主题名称输入框
- 技能列表（可删除）
- 添加技能下拉框
- 保存/取消按钮

### 删除主题

点击"删除"按钮，显示确认：
- 简单的 `confirm()` 弹窗
- 如果主题正在使用中，提示用户先切换到其他主题

## 简化总结

| 维度 | 旧设计 | 新设计 |
|------|--------|--------|
| 页面数 | 4（Setup, Dashboard, Skills, Settings） | 2（Setup, Dashboard） |
| 核心操作点击数 | 4 | 2 |
| 需理解概念 | 4（仓库、目标、主题、符号链接） | 1（主题） |
| 初始化步骤 | 3 | 1 |
| 技能页 | 独立页面 | 删除（合并到主页） |
| 设置页 | 独立页面 | 删除（所有设置在主页） |
| 目标目录 | 初始化必填 | 有默认值，可选 |
| 主题切换 | 全局统一 | 每个目录独立 |
| 技能仓库 | 设置页配置 | 主页直接编辑 |
