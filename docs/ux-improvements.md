# Skill Switch 用户体验优化计划

## 已完成优化 (P0/P1)

### P0: 概念理解障碍
**问题**：用户需要理解 4 个抽象概念（仓库目录、目标目录、主题、符号链接）

**解决方案**：
- 将"仓库目录"改为"技能库"
- 将"目标目录"改为"工具目录"
- 将"主题"改为"技能组合"
- 将"全量"改为"完整版"
- 添加解释性文案

### P0: 初始化体验
**问题**：初始化向导要求用户立即配置两个目录，但用户不理解它们的作用

**解决方案**：
- 添加解释文案："存放所有技能文件的地方"
- 添加解释文案："Claude Code、OpenCode 等工具读取技能的位置"
- 优化预览文案："将创建'完整版'技能组合，包含以下所有技能"

### P1: Dashboard 信息架构
**问题**：技能仓库区域占据顶部黄金位置，但只是显示路径，很少需要修改

**解决方案**：
- 将技能库区域改为可折叠的
- 点击标题区域可展开/折叠技能列表
- 显示技能数量，点击可查看详情

### P1: 核心操作路径
**问题**：切换主题的下拉框位置不直观

**解决方案**：
- 重新布局工具目录卡片
- 将当前主题显示和切换下拉框放在右侧
- 优化视觉层级

## 待优化内容 (P2/P2)

### P2: 主题编辑体验问题

**问题描述**：
主题编辑采用"卡片内联展开"模式，存在多个问题。

**具体问题**：
1. **展开后卡片高度突变**，挤压其他卡片
2. **添加技能的交互**：点击"添加技能"后显示一个下拉框，但下拉框会消失
3. **没有撤销功能**：删除技能后需要重新添加
4. **保存/取消按钮位置**：在卡片底部，需要滚动

**用户困惑**：
> "我删除了一个技能，怎么撤销？"
> "添加技能的下拉框怎么消失了？"
> "保存按钮在哪？"

**解决方案**：
1. **使用 Modal 编辑**（而非内联展开）：
   ```
   ┌─────────────────────────────────────┐
   │  编辑技能组合：开发                   │
   ├─────────────────────────────────────┤
   │                                      │
   │  名称：[开发                        ]│
   │                                      │
   │  包含的技能：                         │
   │  ┌─────────────────────────────────┐ │
   │  │  skill-a  [×]                   │ │
   │  │  skill-b  [×]                   │ │
   │  │  skill-c  [×]                   │ │
   │  └─────────────────────────────────┘ │
   │                                      │
   │  可添加的技能：                       │
   │  [skill-d] [skill-e] [skill-f]       │  ← 点击添加
   │                                      │
   │              [取消]  [保存]           │
   └─────────────────────────────────────┘
   ```

2. **实时预览**：
   - 编辑时实时显示"修改后：X 个技能"
   - 删除技能时显示确认提示（而非直接删除）

3. **技能选择优化**：
   - 使用 Checkbox 列表，而非"删除 + 添加"模式
   - 支持搜索/筛选技能

### P2: 删除操作体验问题

**问题描述**：
删除目录、删除主题都使用浏览器原生 `confirm()` 弹窗。

**问题**：
1. 原生弹窗样式丑陋，与应用风格不一致
2. 弹窗文案过于简单，没有说明后果
3. 没有"撤销"功能

**用户困惑**：
> "删除目录会影响什么？"
> "删除主题后，正在使用这个主题的目录会怎样？"
> "我能撤销吗？"

**解决方案**：
1. **自定义确认 Modal**：
   ```
   ┌─────────────────────────────────────┐
   │  删除工具目录                         │
   ├─────────────────────────────────────┤
   │                                      │
   │  确定要删除这个工具目录吗？             │
   │  ~/.claude/skills                    │
   │                                      │
   │  ⚠️ 这将移除该目录下的所有符号链接。    │
   │  原始技能文件不会被删除。               │
   │                                      │
   │              [取消]  [删除]           │
   └─────────────────────────────────────┘
   ```

2. **删除主题的特殊处理**：
   ```
   ┌─────────────────────────────────────┐
   │  删除技能组合：开发                   │
   ├─────────────────────────────────────┤
   │                                      │
   │  ⚠️ 以下工具目录正在使用这个组合：      │
   │  • ~/.opencode/skills                │
   │                                      │
   │  删除后，这些目录将保持当前状态，       │
   │  但无法再切换到"开发"组合。            │
   │                                      │
   │              [取消]  [删除]           │
   └─────────────────────────────────────┘
   ```

### P3: 状态反馈问题

**问题描述**：
操作后的反馈不够及时和明确。

**具体问题**：
1. **切换主题后**：只有 Toast 提示，卡片状态没有立即更新
2. **保存编辑后**：需要等待 API 响应，没有 loading 状态
3. **错误处理**：错误信息过于技术化（如 "ENOENT"）

**用户困惑**：
> "切换成功了吗？我怎么确认？"
> "保存按钮按了没反应？"
> "报错信息看不懂"

**解决方案**：
1. **即时状态更新**：
   - 切换主题后，立即更新卡片的"当前主题"显示
   - 添加 loading 动画（如骨架屏）

2. **操作反馈优化**：
   ```
   保存中...  →  ✓ 已保存  →  （2 秒后消失）
   ```

3. **错误信息人性化**：
   ```
   技术错误：ENOENT: no such file or directory
   用户提示：目录不存在，请检查路径是否正确
   ```

### P3: 移动端体验问题

**问题描述**：
虽然有响应式布局，但移动端体验仍有问题。

**具体问题**：
1. 下拉框在移动端全宽，但选项文字可能被截断
2. 卡片内的标签列表在小屏幕上会换行多次，高度不可控
3. 编辑模式下的输入框和按钮堆叠，操作繁琐

**解决方案**：
1. **移动端专用布局**：
   - 卡片改为垂直堆叠（已实现）
   - 下拉框改为底部弹出的选择器
   - 标签列表限制行数，显示"+N"按钮展开

2. **触摸优化**：
   - 按钮增大到 44px 高度（符合触摸规范）
   - 添加适当的间距，避免误触

### P3: 高级功能缺失

**问题描述**：
缺少一些提升效率的高级功能。

**建议添加**：
1. **主题预设**：提供常用主题模板（如"前端开发"、"全栈"、"演示"）
2. **导入/导出**：支持导出主题配置，方便团队共享
3. **快捷切换**：支持键盘快捷键（如 Ctrl+1/2/3 切换主题）
4. **历史记录**：显示最近使用的主题，快速回退

### P3: 文案和国际化问题

**问题描述**：
部分文案不够清晰，且没有完整的中文支持。

**具体问题**：
1. "技能仓库" → 应该是"技能库"或"技能源" ✅ 已完成
2. "目录与主题" → 语义不清，应该是"工具目录" ✅ 已完成
3. "全量" → 太抽象，应该是"完整版" ✅ 已完成
4. 错误信息部分是英文，部分是中文

**解决方案**：
1. **统一术语**：
   - 仓库目录 → "技能库" ✅ 已完成
   - 目标目录 → "工具目录" ✅ 已完成
   - 主题 → "技能组合" ✅ 已完成
   - 全量 → "完整版" ✅ 已完成

2. **完整中文翻译**：所有 UI 文案、错误信息、提示信息

## 总结

当前实现已经按照 `docs/redesign.md` 进行了简化，P0/P1 优化已完成：

1. **概念层面**：将技术术语转化为用户语言 ✅
2. **交互层面**：核心操作路径可以更短、更直观 ✅
3. **视觉层面**：信息层级需要重新梳理 ✅
4. **反馈层面**：操作反馈需要更及时、更明确

最关键的是**降低认知负荷**：用户不需要理解"符号链接"、"仓库目录"、"目标目录"这些技术概念，只需要知道"我要为 Claude Code 选择一套技能组合"。
