# Skill Switch 设计问题分析报告

## 核心问题：这是一个"工程师设计"的界面

当前设计最大的问题是**用技术概念代替了用户心智模型**。用户看到的是：
- 一个"技能库"卡片（不知道是干嘛的）
- 一个"工具目录"列表（每个卡片布局混乱）
- 一个"技能组合"网格（不知道和上面的关系）

---

## 问题 1: 视觉层级混乱

**现状**：
```
┌─────────────────────────────────────────┐
│  技能库                              [编辑] │  ← 28px 标题 + 描述
│  ~/Documents/github-skills              │
│  9 个技能                               │
└─────────────────────────────────────────┘
        ↓ gap: 28px
┌─────────────────────────────────────────┐
│  工具目录                          [+添加]│  ← 28px 标题（page-title）
│  每个工具目录可以独立绑定一套技能组合。    │
└─────────────────────────────────────────┘
        ↓ gap: 12px
┌─────────────────────────────────────────┐
│  ~/.claude/skills     [编辑][删除]       │  ← 卡片内布局混乱
│  完整版  9 个技能    [下拉框]            │
│  skill-a  skill-b  skill-c  ...         │
└─────────────────────────────────────────┘
        ↓ gap: 28px
┌─────────────────────────────────────────┐
│  技能组合                          [+新建]│  ← 18px 标题（section-title）
│  维护技能组合的名称和包含的技能。         │
└─────────────────────────────────────────┘
        ↓ gap: 12px
┌──────┐ ┌──────┐ ┌──────┐
│ 完整版│ │ 开发  │ │ 设计  │
│ 9 技能│ │ 3 技能│ │ 2 技能│
└──────┘ └──────┘ └──────┘
```

**问题**：
1. "工具目录"用 28px 大标题，"技能组合"用 18px 小标题 → **权重不一致**
2. 技能库区域占据顶部黄金位置，但它是**低频操作**
3. 三个区域之间没有明确的**视觉关联**，用户不知道它们的关系

---

## 问题 2: 工具目录卡片布局混乱

**现状**：
```
┌─────────────────────────────────────────────────────────────┐
│  ~/.claude/skills                    [编辑] [删除]          │  ← 路径 + 按钮挤在一起
│                                  完整版  9 个技能  [下拉框] │  ← 主题信息在右侧，远离路径
│  skill-a  skill-b  skill-c  skill-d  ...                   │  ← 技能标签在底部
└─────────────────────────────────────────────────────────────┘
```

**问题**：
1. **路径和操作按钮分离**：用户看到路径，但操作按钮在右边，需要眼睛跳跃
2. **主题信息位置奇怪**：当前主题显示在右上角，但用户期望它在路径下面
3. **下拉框位置不直观**：用户需要切换主题，但下拉框在右上角，远离主题名称
4. **视觉重心分散**：左侧是路径，右侧是主题，中间是空白

**用户困惑**：
> "我要切换主题，但下拉框在哪？"
> "当前主题是'完整版'，但它显示在右边，我不确定这是什么意思"

---

## 问题 3: 信息架构不清晰

**现状**：三个区域并列，没有明确的从属关系

**用户心智模型**：
```
我有多个工具（Claude Code、OpenCode、Codex）
每个工具需要一套技能组合
技能组合是从技能库里选出来的
```

**当前设计**：
```
技能库（顶部）
工具目录（中间）
技能组合（底部）
```

**问题**：
- 用户不知道"技能库"和下面两个区域的关系
- "工具目录"和"技能组合"看起来是并列的，但实际上有从属关系
- 用户需要**滚动**才能看到技能组合，但它是核心功能

---

## 问题 4: 卡片设计缺乏层次

**现状**：所有卡片都是白底 + 浅灰边框 + 微弱阴影

**问题**：
1. **所有卡片视觉权重相同**：技能库卡片、工具目录卡片、技能组合卡片看起来一样重要
2. **缺乏视觉层次**：没有主次之分，用户不知道先看哪里
3. **卡片内部缺乏层次**：路径、主题、技能标签都是平级的

**对比参考**（Linear、Stripe 等优秀设计）：
- 主要卡片有更强的阴影或边框
- 次要卡片更轻量
- 卡片内部有明确的信息层级

---

## 问题 5: 按钮系统混乱

**现状**：
- 顶部：编辑按钮（btn-secondary btn-sm）
- 工具目录：编辑、删除按钮（btn-secondary btn-sm、btn-danger btn-sm）
- 添加工具目录：主按钮（btn-primary）
- 新建技能组合：主按钮（btn-primary）
- 技能组合卡片：编辑、删除按钮（btn-secondary btn-sm、btn-danger btn-sm）

**问题**：
1. **按钮位置不一致**：有时在卡片头部，有时在卡片底部
2. **按钮大小不一致**：有 btn-sm，有普通 btn
3. **按钮颜色不一致**：有 primary、secondary、danger，但使用场景不明确
4. **操作入口分散**：添加/新建按钮在 section 头部，编辑/删除在卡片内部

---

## 问题 6: Tag 标签缺乏语义

**现状**：
```html
<span className="tag">skill-name</span>
<button className="tag clickable">skill-name</button>
```

**问题**：
1. **可点击和不可点击的标签视觉相同**：用户不知道哪些可以点
2. **标签没有语义**：所有技能都是灰色标签，没有分类、状态、优先级
3. **标签列表溢出处理**：只显示前 5 个，显示"+N"，但用户不知道 N 是什么

---

## 问题 7: 响应式设计不完善

**现状**：
```css
@media (max-width: 720px) {
  .section-header,
  .card-header,
  .target-card-header {
    align-items: stretch;
    flex-direction: column;
  }
}
```

**问题**：
1. **断点选择不合理**：720px 对于桌面端来说太窄，对于移动端来说太宽
2. **布局突变**：从水平布局突然变成垂直布局，没有过渡
3. **触摸目标太小**：按钮在移动端仍然是 28px 高度，不符合 44px 最小触摸目标

---

## 问题 8: 缺乏视觉反馈

**现状**：
- 切换主题后只有 Toast 提示
- 编辑模式下卡片变成 edit-card（左边框蓝色）
- 删除使用原生 confirm() 弹窗

**问题**：
1. **操作反馈不及时**：用户点击后需要等待 API 响应
2. **状态变化不明显**：编辑模式和查看模式视觉差异小
3. **删除确认体验差**：原生弹窗与应用风格不一致

---

## 问题 9: 缺乏空状态设计

**现状**：
```html
<div className="empty-state">还没有工具目录</div>
```

**问题**：
1. **空状态太简单**：只有一行文字，没有引导用户下一步操作
2. **没有视觉元素**：没有图标、插图，显得冷冰冰
3. **缺乏行动号召**：没有"立即添加"按钮

---

## 问题 10: 缺乏品牌感

**现状**：
- 顶部栏只有 "SS" 标志和 "Skill Switch" 文字
- 颜色系统是标准的蓝色 + 灰色
- 没有独特的视觉元素

**问题**：
1. **缺乏个性**：看起来像任何管理后台
2. **没有情感连接**：用户不会对这个界面产生好感
3. **缺乏记忆点**：没有独特的视觉符号

---

## 总结：设计方向建议

### 1. 重新定义信息架构
```
┌─────────────────────────────────────────┐
│  Skill Switch                           │
├─────────────────────────────────────────┤
│                                          │
│  [技能库] ← 折叠显示，点击展开          │
│                                          │
│  ┌─────────────────────────────────────┐│
│  │  Claude Code                        ││  ← 每个工具一个大卡片
│  │  ~/.claude/skills                   ││
│  │  当前：完整版（9 个技能）            ││
│  │  [切换技能组合]                      ││
│  │  skill-a  skill-b  skill-c  ...     ││
│  └─────────────────────────────────────┘│
│                                          │
│  ┌─────────────────────────────────────┐│
│  │  OpenCode                           ││
│  │  ~/.opencode/skills                 ││
│  │  当前：开发（3 个技能）              ││
│  │  [切换技能组合]                      ││
│  │  skill-a  skill-d                   ││
│  └─────────────────────────────────────┘│
│                                          │
│  [技能组合] ← 底部，管理功能            │
│  ┌──────┐ ┌──────┐ ┌──────┐            │
│  │ 完整版│ │ 开发  │ │ 设计  │            │
│  └──────┘ └──────┘ └──────┘            │
└─────────────────────────────────────────┘
```

### 2. 统一卡片设计
- 每个工具目录一个大卡片，内部结构清晰
- 技能组合卡片更轻量，作为管理功能

### 3. 突出核心操作
- 切换主题是核心操作，放在卡片显眼位置
- 编辑/删除是次要操作，放在卡片底部或右侧

### 4. 建立视觉层次
- 主要卡片更强的阴影/边框
- 次要卡片更轻量
- 卡片内部有明确的信息层级

### 5. 统一按钮系统
- 主要操作：primary 按钮
- 次要操作：secondary 按钮
- 危险操作：danger 按钮
- 位置一致：操作按钮在卡片底部或右侧

### 6. 增强反馈系统
- 即时状态更新
- 自定义确认弹窗
- 加载状态

### 7. 完善响应式
- 合理断点
- 触摸友好
- 渐进增强

### 8. 增加品牌感
- 独特的视觉元素
- 情感化设计
- 记忆点
